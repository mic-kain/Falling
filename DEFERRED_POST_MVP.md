# Falling — Deferred Post-MVP Scope

Status: Locked. These items must not be implemented until the Tier 1 core loop has been validated through playtesting and instrumentation.

## 17. Post-MVP Deferred Scope

| Category | Deferred features |
|---|---|
| Procedural generation | Angled, narrow-class, wide recovery, fragile, stable-looking or moving platform types; wind; moving hazards; optional routes; risk/reward scoring; difficulty budgets; full ten-stage validation; branch reconnection. |
| Progression | Cosmetics, skins, trails, environments, currencies, achievements, missions, levels or daily challenges. |
| Competitive/social | Leaderboards, Game Center, friends, ghost runs, sharing or seeded competitions. |
| Monetisation | Ads, purchases, subscriptions, paid revives, offer codes or shops. |
| Content | Enemies, combat, power-ups, narrative, multiple game modes, biomes or advanced adaptive music. |
| Platforms | Android, iPad-specific version, macOS, TV, console, Vision Pro or cross-platform migration. |

## Summary from Tier 1 Scope

The following is explicitly out of scope for Tier 1 and is restated here for a single source of truth:

- All Tier 2 procedural generation: angled platforms, special platform classes, moving platforms, wind, moving hazards, branching routes, optional risk/reward routes, difficulty budgets and the full ten-stage validation pipeline.
- Cosmetics, character skins, unlockable environments and progression systems.
- Monetisation, advertisements, subscriptions, paid continues or revives.
- Leaderboards, Game Center, achievements, daily challenges, social sharing or ghost runs.
- Android, iPad-specific layouts, macOS, console or any cross-platform engine work.
- Enemies, combat, health, checkpoints, story, multiple modes or power-ups.

## Guiding Principle

Tier 2 complexity must not be added to disguise weaknesses in the core mechanic. The Tier 1 core loop must first be validated against its locked success metrics before any of the above is considered.
